# 商品详情素材预览状态带设计稿

- 日期：2026-04-03
- 主题：Asset Preview Status Strip
- 项目：`next-shadcn-admin-dashboard`
- 目标：在商品详情页右侧素材预览卡头部增加一个轻量状态带，让渠道字段保存后，用户能立刻看到“已保存 / 当前仍缺几项 / 最近保存时间”这类反馈，但不改变图片预览内容本身。

## 1. 背景

当前详情页已经具备两条基础能力：

1. 渠道字段可编辑并保存到共享 store
2. 右侧素材预览会跟随当前渠道 `Tab` 切换

这已经能完成“编辑字段”和“看不同渠道素材节奏”两件事，但两者之间还没有轻量联动。

结果是：

1. 用户点了“保存字段”后，右侧素材卡看起来几乎没反馈
2. 用户很难第一眼判断“这次保存到底有没有让当前渠道变得更完整”
3. 如果不想把素材卡做成复杂工作台，就需要一种更轻、更贴近当前布局的反馈方式

因此这一轮不去改 preview 图片本身，而是在素材卡头部增加一个轻量状态带。

## 2. 已确认决策

### 2.1 联动深度

本轮只做“头部提示联动”，不做图片内容联动。

也就是：

- 保存后，素材卡头部状态变化
- 图片内容本身不因为字段保存而立即换图或变文案

明确不做：

- 预览图内容变化
- 局部素材段内部 badge 批量联动
- 全局 toast 体系扩展

### 2.2 反馈形态

采用 `badge 组合状态带`，不采用提示卡或长段说明文案。

原因：

1. 最适合当前后台卡片风格
2. 扫读快
3. 不会压过封面图和组图

### 2.3 展示信息

状态带只展示 3 类信息：

1. 保存反馈
2. 当前渠道缺失情况
3. 最近保存时间

不展示：

- 审核状态
- 上下架状态
- 渠道动作按钮
- 额外解释性长文案

### 2.4 状态来源

本轮不新增新的全局业务状态源。

状态分成两类来源：

1. `持久业务状态`
   - 来自当前 `product.channels[channelId]`
   - 包含：
     - `missingFields`
     - `lastSyncAt`
2. `瞬时保存反馈`
   - 来自详情页本地状态
   - 只表达“当前渠道刚刚保存成功过”

## 3. 目标界面

### 3.1 位置

状态带放在素材卡标题区下方，也就是：

1. `CardTitle`
2. `CardDescription`
3. `status badges`
4. 正文预览内容

### 3.2 表现形式

状态带最多展示 3 个 badge：

1. 保存反馈 badge
2. 当前渠道缺失情况 badge
3. 最近保存时间 badge

例如：

- `已保存`
- `视频号缺 2 项`
- `最近 09:40`

或在未刚保存时：

- `最近保存记录`
- `抖音已齐备`
- `最近 09:55`

### 3.3 视觉要求

状态带必须是“轻量补充信息”，不能升级成第二个状态面板。

要求：

- badge 数量固定且少
- 不加大块背景条
- 不增加多行文案解释
- 不在视觉权重上盖过封面图

## 4. 数据与联动设计

### 4.1 当前渠道缺失情况

直接读取：

```ts
product.channels[channelId].missingFields.length
```

渲染规则：

- `0` => `已齐备`
- `> 0` => `缺 X 项`

这里表达的是“当前渠道字段是否齐备”，不是“素材是否齐备”。

### 4.2 最近保存时间

直接读取：

```ts
product.channels[channelId].lastSyncAt
```

本轮不区分“真实同步时间”和“本地 mock 保存时间”，沿用当前详情页已有语义。

### 4.3 保存反馈

建议在 `ProductDetailPage` 中维护一个本地瞬时状态，例如：

```ts
lastSavedChannelId?: ChannelId
```

保存成功时：

- 如果当前保存的是 `douyin`，则标记 `lastSavedChannelId = "douyin"`
- 如果当前保存的是 `wechat`，则标记 `lastSavedChannelId = "wechat"`

素材卡拿到：

- `channelId`
- `didSave`

之后按当前渠道决定显示：

- 当前渠道刚保存过 => `已保存`
- 否则 => `最近保存记录`

本轮不要求为每个渠道单独维护复杂历史。

## 5. 前端实现建议

### 5.1 状态提升位置

建议仍然把瞬时保存反馈放在 `ProductDetailPage`，不要塞进 store。

原因：

1. 它不是业务主数据
2. 它只服务当前详情页局部反馈
3. 不值得污染全局共享数据层

### 5.2 接口变化

建议 `AssetsCard` 新增一个轻量状态入参，例如：

```ts
statusStrip: {
  didSave: boolean;
  missingCount: number;
  lastSavedAt: string;
}
```

或者拆成三个平铺 props 也可以，但建议收成一个对象，语义更清楚。

### 5.3 保存回调联动

当前 `ProductDetailPage` 里已经有：

```ts
const handleSave = (...)
```

本轮只需要在成功保存后额外更新本地状态：

1. 记录刚保存的是哪个 `channel`
2. 由当前 `product.channels[channelId]` 驱动缺失项和时间刷新

不需要改 `saveDraft` 的 store 契约。

## 6. 成功标准

本次完成后，应满足以下条件：

1. 用户保存当前渠道字段后，右侧素材卡头部状态带立即变化。
2. 切换渠道时，状态带跟随当前渠道数据切换。
3. 状态带足够轻，不抢过图片预览主体。
4. 不新增第二套业务状态源，仍然复用现有详情页与 store 数据。

## 7. 后续衔接

如果这轮轻量联动成立，下一步可以继续选：

1. 让素材卡内部某些段落 badge 也跟随缺失状态变化
2. 让 preview mock 与特定字段类型建立更明确映射
3. 把保存反馈扩展成列表页或工作台层面的统一回执提示

本次设计只覆盖“素材卡头部状态带联动”，不预支更强的 preview 内容联动。
